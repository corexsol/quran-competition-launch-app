# King Abdulaziz Quran Competition Launch App - Design

## Objective

Build a ceremony-safe, two-screen launch experience for a ninth-generation iPad in landscape orientation. The only interaction is a single tap on the Arabic Start button, followed by a dignified transition to the competition statistics. The app must run fully offline after one successful online load and installation to the iPad home screen.

## Source Authority

The target device and ceremony flow in the written brief are authoritative. The supplied PDFs are an artwork library rather than a literal filename or page-layout map because the two-page master is 16:9 while the target iPad is 4:3, and several supplied filenames contain artwork different from the brief's labels.

The app will therefore use the artwork by inspected visual content:

- `Asset 1.pdf`: circular Arabic Start button.
- `Asset 2.pdf`: minister supervision text.
- `Asset 3.pdf`: primary launch title artwork.
- `Asset 4.pdf`: pale side ornaments.
- `Asset 5.pdf`: pale central medallion.
- `Asset 99.pdf`: alternate title render, retained only as a fallback if its final high-resolution conversion is visibly cleaner.
- `Asset 1022.pdf`: cycle, country, participant, and judging statistics.
- `Artboard 44.pdf`: unrelated general competition infographic; excluded from the two-screen app.
- `تطبيق التدشين.pdf`: visual tone reference only; not reproduced literally because it is a green 16:9 composition.

## Visual System

The app uses a fixed 4:3 composition designed at 1080 x 810 CSS pixels and scales proportionally to the actual landscape viewport. Both screens share a white background, pale green side ornaments, and a low-opacity central medallion. Gold and deep competition green remain the only strong colors. Critical Arabic typography stays baked into the supplied artwork, avoiding font-loading risk and preserving the calligraphy exactly.

Screen 1 is a centered vertical composition:

1. Primary title artwork in the upper portion.
2. Minister supervision artwork directly below it.
3. Circular Start button in the lower portion, at least 300 CSS pixels wide at the design viewport.

Spacing will preserve a clear hierarchy and keep all artwork inside an iPad safe area. The button remains visually dominant without overlapping the text artwork.

Screen 2 centers the `Asset 1022` statistics artwork at the largest size that fits without cropping. The shared background keeps the transition visually continuous. Because the statistics are supplied as a single composed infographic, it will animate as one unit rather than being cut into independently moving pieces; this preserves layout fidelity and eliminates seam risk.

## Asset Preparation

Each included PDF is rendered to a local transparent PNG at approximately twice its maximum displayed size. White PDF page backgrounds are converted to alpha only where they are clearly outside the artwork; shadows and near-white highlights inside the artwork are preserved. Output dimensions are trimmed to meaningful content bounds with a small safety margin.

The final asset set includes:

- primary title PNG;
- minister text PNG;
- Start button PNG;
- side ornament PNG;
- central medallion PNG;
- statistics PNG;
- 180 x 180 Apple touch icon;
- 192 x 192 and 512 x 512 manifest icons.

Icons use a clean crop of the competition medallion from the supplied source artwork on a deep green field. PNGs are losslessly optimized while keeping the total application payload below 10 MB.

## Interaction and Transition

The Start button is a real HTML `button` containing the supplied artwork. A single `click` listener handles the first activation generated by touch, pointer, or keyboard input:

1. JavaScript immediately marks the app as launched, disables the button, and adds the transition state class.
2. The button scales to about 0.96 for roughly 100 ms.
3. A gold radial glow, centered on the button, expands with transform and opacity only.
4. Screen 1 fades out while Screen 2 becomes visible beneath the glow.
5. The statistics artwork fades and slides upward slightly as the glow dissipates.

Total transition duration is about 1.9 seconds. Repeated taps and synthetic follow-up clicks are ignored after the first activation. Once Screen 2 is shown, the app remains there until it is reloaded or relaunched; there is no back control or routing.

The app respects `prefers-reduced-motion` with a short cross-fade while preserving the same state change.

## Application Structure

The deliverable is a plain static app rooted at `launch-app/`:

- `index.html`: semantic two-screen DOM, metadata, local asset preloads, manifest and iOS links.
- `style.css`: fixed-stage layout, ceremony hardening, artwork placement, and all transition animation.
- `app.js`: one-shot interaction state, image readiness safeguards, and service-worker registration.
- `sw.js`: versioned precache and cache-first request handling.
- `manifest.json`: standalone landscape PWA metadata and icons. It uses the Arabic name `تطبيق تدشين مسابقة الملك عبدالعزيز الدولية`, the short name `تطبيق التدشين`, theme color `#1a4d2e`, and a white background.
- `assets/`: converted PNG artwork and icons.
- `README.md`: serving, iPad installation, and offline rehearsal instructions.

No framework, router, build process, analytics, external font, or CDN dependency is used.

## Offline and Failure Behavior

The service worker precaches the HTML, CSS, JavaScript, manifest, and every image during installation. Same-origin requests are served cache-first, and navigation requests fall back to cached `index.html`. The service worker activates promptly and removes only caches owned by older versions of this app.

The page also preloads the Screen 2 statistics image during initial startup. The Start button becomes interactive only after all transition-critical imagery is decoded. A service-worker registration failure is logged but never blocks the ceremony interaction, because the currently loaded page remains completely local and functional.

## Ceremony Hardening

The document uses a locked viewport with `maximum-scale=1` and `user-scalable=no`, landscape manifest orientation, `touch-action: manipulation`, hidden overflow, disabled overscroll, disabled selection and callouts, and no scrollbars. Safe-area insets are accounted for. It includes the required iOS standalone, translucent status-bar, and Apple touch-icon metadata. The layout contains no links or controls other than Start, and no focus ring remains visible after touch activation.

The button keeps an accessible Arabic name and remains keyboard-activatable for local testing. Animation uses only opacity and transforms to keep Safari compositing smooth.

## Verification

Implementation is complete only after all of the following pass:

- Every expected file exists and every referenced asset resolves locally.
- The app renders without horizontal or vertical overflow at 1080 x 810 and 1024 x 768.
- Screen 1 appears immediately with sharp artwork and a Start target at least 300 CSS pixels wide at 1080 x 810.
- One activation triggers exactly one transition and lands on Screen 2.
- Rapid repeated taps do not restart or corrupt the transition.
- Screen 2 is fully visible before and after disabling the network.
- A fresh service-worker install precaches the complete asset list, and a reload works with the server unavailable after the cache is established.
- The browser console has no missing-resource, JavaScript, manifest, or service-worker errors.
- The final payload is below 10 MB.
- Rendered screenshots at both target viewport sizes show no clipping, overlap, unintended white boxes, or illegible statistics.

Before the ceremony, the operator must open the installed app once online, confirm Screen 1, close it, enable airplane mode, relaunch it, complete the transition, and repeat the test once more.
